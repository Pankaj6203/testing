Prerequisites

Make sure you have these before starting

  • GitHub account (github.com)
  • cPanel hosting account (MilesWeb, Hostinger, Namecheap, etc.)
  • Git installed on your PC — check with git --version
  • A code editor (VS Code recommended)
  • Your cPanel username (shown on cPanel dashboard)
1

Create GitHub Repository

Set up the remote repository on GitHub

  1. Go to github.com and log in
  2. Click "+" → "New repository"
  3. Enter Repository name (e.g. myproject)
  4. Set to Public or Private
  5. Do NOT check "Add a README file"
  6. Click "Create repository"
  7. Copy the HTTPS URL — e.g. https://github.com/username/myproject.git
⚠ Important

Do not initialize with README, .gitignore, or license. Keep it completely empty so it doesn't conflict with your local project.

2

Create Local Project with .cpanel.yml

Set up your project folder with the deployment config

Project folder structure

myproject/
  ├── .cpanel.yml   ← deployment config (required)
  ├── .gitignore
  ├── index.php
  └── style.css

The .cpanel.yml file (most important file)

--- 
deployment:
  tasks:
    - export DEPLOYPATH=/home/YOUR_USERNAME/your-subdomain.com
    - /bin/cp -R . $DEPLOYPATH
ℹ Replace these values

YOUR_USERNAME → your cPanel username (e.g. zkakdxqb)
your-subdomain.com → the folder name of your subdomain (e.g. ci.techotd.in)

⚠ Never use $CPANEL_BASE_INSTALL_PATH

This variable is undefined and will copy the entire root filesystem into your deployment folder, filling up your disk quota. Always use . (dot) which means current directory.

Initialize Git locally

# Run these in your project folder (PowerShell / Terminal)
git init
git branch -m master main
git add .
git commit -m "Initial commit"
3

Push Project to GitHub

Connect local repo to GitHub and push

# Add GitHub as remote origin
git remote add origin https://github.com/USERNAME/REPO.git

# Push to GitHub (browser will open for login first time)
git push -u origin main
ℹ First-time Authentication

On Windows, Git Credential Manager will open a browser window for GitHub login. Log in once — it saves credentials automatically for future pushes.

After push, verify your files are visible at github.com/USERNAME/REPO

4

Create Subdomain on cPanel

Set up where the site will be served from

  1. Log in to cPanel
  2. Go to Domains → Subdomains (or just "Domains")
  3. Enter subdomain name (e.g. ci) → select your main domain
  4. Document root will auto-fill as public_html/ci — you can change it
  5. Click Create
ℹ Note the document root path

When the subdomain is created, cPanel shows the full path like /home/zkakdxqb/ci.techotd.in. This is the path you put in .cpanel.yml as DEPLOYPATH.

5

Connect GitHub to cPanel Git Version Control

Clone your GitHub repo inside cPanel

  1. In cPanel, search for "Git Version Control" and open it
  2. Click "Create"
  3. Enable "Clone a Repository" toggle
  4. Fill in the form:
Field Value Example
Clone URL Your GitHub HTTPS URL https://github.com/user/repo.git
Repository Path A separate folder (NOT the subdomain folder) repositories/myproject
Repository Name Any display name myproject
⚠ Critical: Repository Path must NOT be your subdomain folder

Use repositories/myproject NOT ci.techotd.in. The .cpanel.yml handles copying files to the subdomain folder. If repo and deploy folder are the same, the copy command will fail.

  1. Click "Create" — cPanel will clone your repo
  2. Wait for "Clone is complete" message
6

Deploy for the First Time

Manually trigger the first deployment

  1. In Git Version Control, click "Manage" next to your repo
  2. Go to "Pull or Deploy" tab
  3. Click "Deploy HEAD Commit"
  4. Wait for deployment to complete
  5. Open your subdomain in browser to verify
✅ Success!

If your site loads correctly, the pipeline is working. The .cpanel.yml copied your files from the repo to your subdomain's document root.

ℹ What "Update from Remote" vs "Deploy HEAD Commit" does

Update from Remote — pulls latest commits from GitHub into cPanel's local copy of the repo. Does NOT deploy files.
Deploy HEAD Commit — runs the .cpanel.yml tasks and copies files to your subdomain folder. This is the actual deploy.

7

Auto-Deploy with GitHub Webhook

Every git push triggers automatic deployment

Currently, after every push you need to manually click "Update from Remote" + "Deploy HEAD Commit" in cPanel. Set up a webhook to automate this.

Get the cPanel Webhook URL

  1. In cPanel Git Version Control, click "Manage"
  2. Under "Basic Information" tab, find the Clone URL section
  3. Look for a webhook URL or use the cPanel API endpoint

Using cPanel's built-in webhook endpoint

# cPanel webhook URL format:
https://YOUR_CPANEL_DOMAIN:2083/cpsess.../frontend/jupiter/
  version_control/index.html#/webhook/REPO_NAME

Add webhook on GitHub

  1. Go to your GitHub repo → Settings → Webhooks
  2. Click "Add webhook"
  3. Paste the cPanel webhook URL in Payload URL
  4. Content type: application/json
  5. Select "Just the push event"
  6. Click "Add webhook"
ℹ Alternative: Use GitHub Actions

If cPanel webhook doesn't work directly, use GitHub Actions with SSH to trigger cPanel deployment. Create a .github/workflows/deploy.yml file in your repo.

GitHub Actions alternative (deploy.yml)

name: Deploy to cPanel
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Deploy via SSH
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.CPANEL_HOST }}
          username: ${{ secrets.CPANEL_USER }}
          password: ${{ secrets.CPANEL_PASS }}
          script: |
            cd ~/repositories/myproject
            git pull origin main
            cp -R . ~/ci.techotd.in/
!

Troubleshooting Common Errors

Fix these issues if something goes wrong

Error Cause Fix
Disk quota exceeded Failed deploy copied system files. Trash folder is full. Run: rm -rf ~/.trash/* in cPanel Terminal
Directory already contains files Old repo folder exists from previous setup Run: rm -rf ~/repositories/myproject
Files deployed to wrong location $CPANEL_BASE_INSTALL_PATH used instead of . Fix .cpanel.yml: use /bin/cp -R . $DEPLOYPATH
could not read Username Git Credential Manager can't open browser in automated terminal Run git push in your regular terminal (not Claude Code terminal)
Site shows old content after deploy cPanel has old version; didn't pull latest Click "Update from Remote" first, then "Deploy HEAD Commit"
Died at VersionControl/Cache.pm Disk quota exceeded — cPanel can't write temp files Free disk space: empty trash, delete unused files

Check disk usage in cPanel Terminal

# Find which hidden folders are large
du -sh ~/.[^.]* 2>/dev/null | sort -rh | head -20

# Empty trash (most common fix)
rm -rf ~/.trash/*

# Check total disk usage
du -sh ~

Daily Workflow After Setup

What you do every time you make changes

Without auto-deploy (manual)

# 1. Make changes to your files locally
# 2. Stage and commit
git add .
git commit -m "description of change"

# 3. Push to GitHub
git push

# 4. Go to cPanel → Git Version Control → Manage
# 5. Click "Update from Remote"
# 6. Click "Deploy HEAD Commit"

With auto-deploy (webhook configured)

# Just these 3 commands — deploy happens automatically!
git add .
git commit -m "description of change"
git push
✅ Pipeline Complete

Your CI/CD pipeline is live. Every push to the main branch will automatically update your website at your subdomain.
